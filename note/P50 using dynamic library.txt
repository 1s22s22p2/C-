动态链接是链接发生在运行时，只有真正启动exe文件才会链接，无法在编译时优化，而静态链接时在编译时发生的
动态链接的一种形式是（静态动态库）虽然动态链接库dll时应该单独的文件但是可执行文件exe知道它的存在，并且把它当作一项需要
另一种形式时我们可以完全动态地加载动态库，这样就和可执行文件没有关系了我们可以启动可执行文件，而它不会要求应该特定的动态库

需要把属性/链接器/输入中的glfw3.lib改为glfw3dll.lib
使用动态数据库要将dll文件复制到exe所在的文件夹中，否则会报找不到dll文件

#include <iostream>
#include <GLFW/glfw3.h>              //若要使用动态链接头文件不用改

//但是对于函数声明有很多不同之处

#define Log(x)	std::cout<<x<<std::endl;

int main()
{
	int a = glfwInit();
	Log(a);
	std::cin.get();
}
